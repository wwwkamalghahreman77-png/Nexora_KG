# وضعیت پروژه شبکه تعرفه مشترک ریلی — بکاپ برای ادامه‌ی کار

اگه این چت محدود/قطع شد، این فایل رو (همراه با کل zip پروژه) به چت جدید
آپلود کن و بگو «از اینجا ادامه بده» - نیازی نیست از اول توضیح بدی.

## کارهایی که تا الان انجام شده

1. **ساختار کامل ربات تلگرامی ساخته شد** (aiogram 3 + SQLAlchemy + SQLite):
   - محاسبه تعرفه از میانگین داده‌های شرکت‌های عضو (`core/tariff_engine.py`)
   - جستجوی فازی نام ایستگاه/کالا با غلط املایی (`core/fuzzy_search.py`)
   - مسیریابی بین دو ایستگاه با گراف + Dijkstra (`core/route_engine.py`)
   - ثبت‌نام شرکت‌ها، توزیع لید، پلن free/pro/enterprise (`core/lead_matching.py`)
   - دستورات ادمین: `/approve`, `/set_tier`, `/add_station`, `/stats`

2. **داده‌ی واقعی از فایل‌های Word کاربر استخراج شد:**
   - ۹ فایل docx آپلود شد (کد ایستگاه‌های ۸ کشور CIS + یک فایل کامل HS کد)
   - کدهای ایستگاه در این فایل‌ها به‌صورت **عکس** بودن (نه متن) - چون سایت
     منبع (railwagonlocation.com) عمداً کدها رو با عکس رندر می‌کنه تا
     اسکرپ نشن
   - با OCR (tesseract + پیش‌پردازش: grayscale → threshold → 8x upscale →
     psm 8) عکس‌ها خونده شدن
   - نتیجه: **۳٬۰۸۲ ایستگاه** (ترکمنستان، تاجیکستان، ازبکستان، قزاقستان،
     قرقیزستان، آذربایجان، بلاروس، روسیه/مسکو) + **۱۲٬۷۹۶ کد کالا/HS**
     (این یکی از جدول متنی تمیز فایل Word استخراج شد، بدون نیاز به OCR،
     دقتش بالاست)

3. **⚠️ نکته‌ی مهم درباره دقت کدهای ایستگاه:**
   OCR روی دو نمونه‌ی شناخته‌شده (سرخس ترکمنستان=756905، عینی
   تاجیکستان=745506) کاملاً درست عمل کرد، اما چون خوندن عکس‌های خیلی کوچیک
   (۴۵×۱۲ پیکسل) هست، احتمال خطای OCR (یک رقم اشتباه) روی بقیه‌ی ۳۰۸۰
   ایستگاه وجود داره. برای همین **همه‌ی این کدها با `code_verified=false`
   وارد دیتابیس شدن** - یعنی ربات به کاربر می‌گه «نیاز به تایید» نه اینکه
   قطعی نشونش بده. قبل از استفاده‌ی تجاری از یک مسیر پرتردد، حتماً کدش رو
   دستی با منبع اصلی چک کن.

4. **دیتابیس واقعی ساخته و پر شد:** `rail_network.db` (داخل zip هست) از
   قبل با این ۳۰۸۲ ایستگاه و ۱۲۷۹۶ کد کالا پر شده - نیازی به اجرای دوباره‌ی
   import نیست، فقط `pip install -r requirements.txt` و `python -m bot.main`.

## کاری که هنوز باقی مونده (مرحله بعد)

- [ ] **مسیرهای ریلی بین ایستگاه‌ها هنوز وارد نشده** (`RailConnection` جدول
      خالیه) - یعنی بخش «مسیریابی» فعلاً چیزی برنمی‌گردونه. باید فایل
      `connections.csv` (کدوم دو ایستگاه مستقیم به‌هم وصلن) رو یا خودت بسازی
      یا داده‌ی بیشتری بدی تا بسازمش.
- [ ] **مسیر جاده‌ای (چندوجهی):** کاربر خواسته اگه مسیر ریلی نبود یا جفتش
      بود، جاده‌ای/ترکیبی هم نشون داده بشه - این فیچر هنوز اضافه نشده،
      نیاز به یک جدول/فیلد جدید برای «road_route» یا «multimodal» داره.
- [ ] نام فارسی و روسی ایستگاه‌ها خالیه (فقط انگلیسی داریم از منبع) - اگه
      لازم داری، باید یا دستی ترجمه بشه یا از منبع دیگه گرفته بشه.
- [ ] هنوز تست واقعی روی سرور تلگرام انجام نشده (اینجا دسترسی اینترنت ندارم).

## فایل‌های کلیدی پروژه

```
rail-tariff-network/
├── rail_network.db          ← دیتابیس آماده و پر شده
├── data_imports/
│   ├── stations.csv         ← ۳۰۸۲ ایستگاه استخراج‌شده
│   └── cargo.csv            ← ۱۲۷۹۶ کد کالا/HS
├── database/models.py       ← Station, CargoType, RailConnection, Partner, Lead, ...
├── database/import_data.py  ← اسکریپت وارد کردن CSV (دوباره لازم نیست اجرا بشه)
├── core/route_engine.py     ← موتور مسیریابی (منتظر داده‌ی connections)
├── core/fuzzy_search.py     ← جستجوی نام با غلط املایی
├── core/tariff_engine.py    ← تخمین تعرفه از میانگین داده شرکت‌ها
└── bot/                     ← کد ربات تلگرام (aiogram)
```
