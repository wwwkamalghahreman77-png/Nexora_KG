# شبکه تعرفه مشترک ریلی CIS (Rail Tariff Network Bot)

ربات تلگرامی رایگان که با تجمیع داده‌های واقعی شرکت‌های حمل‌ونقل عضو، تعرفه
تقریبی حمل ریلی رو محاسبه می‌کنه و لید مشتری رو بین شرکت‌ها توزیع می‌کنه.

---

## ۱. راه‌اندازی فنی (همین امروز)

1. پایتون ۳.۱۱+ نصب کن.
2. `pip install -r requirements.txt`
3. فایل `.env.example` رو کپی کن به `.env` و مقداردهی کن:
   - `BOT_TOKEN` از @BotFather بگیر
   - `ADMIN_IDS` آیدی عددی خودت (از @userinfobot)
4. داده‌های اولیه رو وارد کن: `python -m database.seed_tariffs`
5. ربات رو اجرا کن: `python -m bot.main`
6. تو تلگرام به رباتت `/start` بزن و تست کن.

هیچ هزینه سروری بالایی نداره چون SQLite رایگانه؛ وقتی چند صد شرکت عضو شدن
می‌تونی به PostgreSQL روی یک VPS ارزان مهاجرت کنی (فقط `DATABASE_URL` عوض میشه).

---

## ۲. نقشه راه اجرا (فاز به فاز)

### فاز صفر - آماده‌سازی (هفته ۱)
- [ ] ربات رو دیپلوی کن (یک VPS ساده یا حتی همون سروری که بات کریپتوت روشه)
- [ ] با ۵-۱۰ نمونه واقعی تعرفه که خودت/آشناهات داری، دیتابیس رو دستی پر کن
  (تا از روز اول کاملاً خالی نباشه)
- [ ] یک پیام معرفی کوتاه فارسی + انگلیسی آماده کن برای گروه‌های تلگرامی فورواردرها

### فاز ۱ - جذب ۳ تا ۵ شرکت شریک اولیه (هفته ۲-۳)
- شرکت‌هایی که مستقیماً رقیب هم نیستن (مثلاً هرکدوم روی مسیر/کشور متفاوت
  تخصص دارن) رو انتخاب کن - اعتماد اولیه راحت‌تر جلب میشه.
- بهشون بگو: «عضویت و ثبت داده رایگانه، در ازاش لید مستقیم می‌گیری.»
- این ۳-۵ شرکت رو با `/approve` در ربات تایید کن.

### فاز ۲ - فعال‌سازی فلای‌ویل داده (هفته ۴-۸)
- هر بار شرکتی لید گرفت و بست، ازش بخواه یک نمونه‌ی تعرفه جدید هم ثبت کنه
  (`/submit_tariff`) - این شرط ادامه‌ی دریافت لید رایگان باشه.
- به‌مرور دیتابیس تعرفه‌ها بزرگ میشه و دقت تخمین‌ها بالا میره.
- تبلیغ بات رو به گروه‌های صنفی حمل‌ونقل، لینکدین، و شبکه‌های واردکننده/صادرکننده گسترش بده.

### فاز ۳ - فعال‌سازی درآمد (بعد از حدود ۵۰-۱۰۰ لید ماهانه)
- شرکت‌های free که اعتبار لید رایگانشون (۳ در ماه، قابل تغییر در کد) تموم شده
  رو با `/set_tier` به `pro` ارتقا بده - برای این کار قرارداد/پرداخت دستی
  (کارت به کارت، فاکتور) کافیه، نیازی به درگاه پرداخت آنلاین از روز اول نیست.
- گزینه دوم درآمد: از شرکت‌هایی که می‌خوان اولویت نمایش داشته باشن، حق عضویت
  ماهانه بگیر (مستقل از تعداد لید).

### فاز ۴ - مقیاس (ماه ۳ به بعد)
- افزودن وب‌سایت ساده (لندینگ + همون فرم محاسبه) برای SEO و اعتبار بیشتر
- افزودن گزارش آماری برای شرکت‌های pro/enterprise (پرتقاضاترین مسیرها/کالاها)
- در نظر گرفتن API برای شرکت‌های بزرگ‌تر که می‌خوان موتور رو تو سیستم خودشون امبد کنن

## ۳. ساختار پروژه

```
rail-tariff-network/
├── bot/
│   ├── main.py              # اجرای ربات
│   └── handlers/
│       ├── start.py         # /start و منوی اصلی
│       ├── calculate.py     # فلوی محاسبه تعرفه + ساخت لید
│       ├── partner.py       # ثبت‌نام شرکت + ثبت داده تعرفه
│       └── admin.py         # (re-export از admin/partner_api.py)
├── database/
│   ├── models.py            # جداول: Station, CargoType, Partner, TariffDataPoint, Lead, Subscription
│   ├── db.py                # اتصال دیتابیس
│   └── seed_tariffs.py      # داده اولیه
├── core/
│   ├── tariff_engine.py     # منطق تخمین قیمت از میانگین داده‌ها
│   └── lead_matching.py     # اولویت‌بندی و توزیع لید بین شرکت‌ها
├── admin/
│   └── partner_api.py       # دستورات ادمین: /approve /set_tier /stats
├── requirements.txt
└── .env.example
```

## ۴. دستورات مهم ربات

| دستور | برای چه کسی | کار |
|---|---|---|
| `/start` | همه | منوی اصلی |
| `/submit_tariff` | شرکت‌های عضو | ثبت یک نمونه تعرفه واقعی |
| `/approve <id>` | ادمین | تایید شرکت جدید |
| `/set_tier <id> <free\|pro\|enterprise>` | ادمین | تغییر پلن شرکت |
| `/stats` | ادمین | آمار کلی شبکه |
| `/station` یا دکمه «جستجوی کد ایستگاه/کالا» | همه | جستجوی فازی نام ایستگاه یا کالا (حتی با غلط املایی/نام ناقص) |
| `/add_station <کد> <نام‌فارسی> <کشور>` | ادمین | ثبت یا تایید کد یک ایستگاه |
| `🗺 مسیریابی حمل` (دکمه منو) | همه | مسیر اصلی + جایگزین بین دو شهر/ایستگاه (فقط بر پایه‌ی خطوط ریلی تاییدشده) |

## ۶. وارد کردن داده‌ی واقعی خودت (کدهای ایستگاه، HS، خطوط ریلی)

فایل‌های CSV بساز (فرمت دقیق ستون‌ها داخل `database/import_data.py` نوشته) و اجرا کن:

```
python -m database.import_data --stations stations.csv --cargo cargo.csv --connections connections.csv
```

- **stations.csv** → کد، نام فارسی/انگلیسی/روسی/محلی، کشور، نقش (import/export/both/transit)
- **cargo.csv** → کد HS شش‌رقمی، کد کامل، نام فارسی
- **connections.csv** → یال‌های گراف مسیریابی؛ فقط خطوطی که مطمئنی الان فعالن رو با
  `is_operational=true` بذار - خطوط در دست ساخت (مثل کریدور کاشغر-اوش که تا الان
  کامل نشده) رو نذار یا `false` بذار، وگرنه ربات مسیر غلط پیشنهاد می‌ده.

## ۷. روشن نگه‌داشتن ربات به‌صورت رایگان با GitHub Actions

چون سرور اختصاصی نداری، این پروژه یه ورک‌فلوی خودکار داره
(`.github/workflows/bot_runner.yml`) که ربات رو با زنجیره‌ی خودکار «تقریباً
۲۴ ساعته» روشن نگه می‌داره - کاملاً رایگان، فقط باید ریپازیتوری **Public**
باشه (GitHub Actions برای ریپوی Public نامحدود و رایگانه).

⚠️ این روش **~۹۹٪ آپ‌تایم** میده نه ۱۰۰٪ تضمینی - بین اجراها ممکنه چند
ثانیه تا چند دقیقه وقفه بیفته. برای شروع کافیه.

### مراحل راه‌اندازی (فقط یک‌بار):

1. **ریپو رو Public کن:** تنظیمات ریپو (Settings) → پایین صفحه (Danger
   Zone) → Change visibility → Public

2. **دسترسی نوشتن به Actions رو فعال کن:** Settings → Actions → General →
   پایین صفحه بخش "Workflow permissions" → گزینه‌ی **Read and write
   permissions** رو انتخاب کن → Save

3. **یه Personal Access Token بساز** (برای اینکه ربات بتونه اجرای بعدی
   خودش رو صدا بزنه - محدودیت امنیتی گیت‌هاب اجازه نمیده توکن پیش‌فرض این
   کارو بکنه):
   - برو پروفایل گیت‌هاب → Settings → پایین صفحه‌ی چپ: Developer settings
   - Personal access tokens → Tokens (classic) → Generate new token (classic)
   - اسمی بده (مثلاً rail-bot-restart)، انقضا رو هرچی خواستی بذار
   - تیک بزن روی scope: `repo` و `workflow`
   - Generate token بزن و **کپی کن** (فقط یه‌بار نشونش میده)

4. **سه تا Secret بساز:** Settings ریپو → Secrets and variables → Actions
   → New repository secret، این سه‌تا رو یکی‌یکی اضافه کن:
   - `BOT_TOKEN` → همون توکنی که از BotFather گرفتی
   - `ADMIN_IDS` → آیدی عددی خودت (از @userinfobot)
   - `GH_PAT` → همون Personal Access Token مرحله‌ی قبل

5. **دستی یه‌بار روشنش کن:** تب Actions ریپو → از لیست چپ
   «rail-bot-runner» رو انتخاب کن → دکمه‌ی Run workflow → Run workflow

از این به بعد خودش هر ~۵ ساعت و ۵۰ دقیقه، اجرای بعدی رو صدا می‌زنه و
زنجیروار ادامه پیدا می‌کنه؛ کرون هر ۶ ساعت هم فال‌بکه اگه زنجیره یه‌جا قطع
بشه. برای دیدن لاگ/خطا: تب Actions → روی آخرین اجرا کلیک کن.

## ۵. نکات مهم قبل از عمومی کردن

- **مسئولیت قیمت:** تو پیام تخمین قید شده که "برآورد است، نه فاکتور رسمی" -
  این خط رو حذف نکن، از نظر حقوقی مسئولیت رو مشخص می‌کنه.
- **حریم خصوصی شماره تماس شرکت‌ها:** فعلاً لید رو ربات به شرکت پوش می‌کنه
  (نه برعکس) که شماره مشتری نهایی لو نره تا خودش تصمیم بگیره با کی صحبت کنه.
- **کدهای ایستگاه/کالا:** لیست `SEED_STATIONS` و `SEED_CARGO` در
  `database/seed_tariffs.py` تمام ۷ مرز رسمی ریلی ایران (سرخس، اینچه‌برون،
  لطف‌آباد، شمتیغ، جلفا، رازی، میرجاوه) رو به اسم دارن، اما **فقط کدهایی که
  خودت تایید کردی (سرخس‌ترکمنستان و عینی‌تاجیکستان) عدد دارن.** بقیه عمداً
  `code=None` گذاشته شدن چون کد عددی رسمی این ایستگاه‌ها (سیستم OSJD/CIS)
  در منابع عمومی وب قابل تایید نبود و حدس زدنش می‌تونه باعث اشتباه مالی/
  لجستیکی واقعی بشه. با `/add_station` (فقط ادمین) هر وقت کد رسمی رو از
  خود راه‌آهن یا شرکت‌های عضو گرفتی، اضافه/تاییدش کن.
- **کدهای HS کالا:** فقط سطح ۶ رقمی جهانی (استاندارد و قابل‌اتکا) وارد شده؛
  کد کامل داخلی/ETSNG هر کشور رو باید شرکت‌های عضو یا منبع رسمی تکمیل کنن.
- **جستجوی فازی:** بخش جدید «جستجوی کد ایستگاه/کالا» با نام ناقص یا غلط
  املایی هم کار می‌کنه (مثلاً کاربر «آهنگران» بنویسه، نزدیک‌ترین نتیجه‌ها
  رو با نام فارسی/انگلیسی/محلی و کد (یا وضعیت "نیاز به تایید") نشون میده.
